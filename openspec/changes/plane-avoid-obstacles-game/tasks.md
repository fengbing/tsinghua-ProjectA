## 总体说明

围绕“可控的飞机 + 随机方块障碍地图 + 碰撞失败反馈”拆解实现任务。任务按推荐顺序排列。

**场景与预制体搭建**：代码已全部就绪，Unity 内的场景、飞机预制体、UI 等需在编辑器中按 **Assets/PlaneGame_场景搭建说明.md** 操作。

## 任务列表

- [ ] **项目与场景准备**
  - [ ] 在 Unity 中创建用于飞机游戏的独立场景（如 `Scenes/PlaneGame.unity`）。
  - [ ] 在场景中添加主摄像机和方向光，调整为适合跟随飞机的 3D 视角。
  - [ ] 创建一个用于放置运行时生成地图和障碍物的空父对象（如 `LevelRoot`）。

- [ ] **飞机预制体与基础外观**
  - [ ] 创建或导入一个飞机模型（可以使用简单的组合体代替，如多个 `Cube`/`Cylinder`）。
  - [ ] 为飞机添加 `Rigidbody` 组件，设置合适的质量、拖拽和重力选项。
  - [ ] 将飞机保存为预制体（如 `Prefabs/Plane`），用于场景实例化和复用。

- [x] **飞机飞行控制脚本（ASWD 控制 + 拟真手感）**
  - [x] 在 `Assets/Scripts` 下创建单独的 C# 控制脚本（如 `PlaneController.cs`）。
  - [x] 在脚本中实现基于 `Rigidbody` 的力 / 力矩控制，而非简单的 `transform.Translate`。
  - [x] 使用键盘 `W/A/S/D` 控制：俯仰、横滚、偏航和推力（“魔法力”），可调节灵敏度参数。
  - [x] 实现基本的重量感和惯性：包括持续重力、速度上限、缓慢的回正效果。
  - [ ] 将控制脚本挂载到飞机预制体上，并在场景中调试参数直到手感合理。

- [x] **地图与障碍物生成（Cube 随机 height/width 地图）**
  - [x] 在 `Assets/Scripts` 中创建地图生成脚本（如 `CubeMapGenerator.cs`），挂到 `LevelRoot` 或单独的管理对象上。
  - [x] 通过参数控制地图整体尺寸（height、width、长度 / 段数）和随机种子。
  - [x] 使用 `Cube` 程序化生成障碍物：如随机高度的墙体、柱子或“门洞”通道。
  - [x] 确保地图生成时为飞机留出合理的可飞行通道，避免完全堵死。
  - [x] 支持每次重新开始时重新生成不同布局的障碍地图。

- [x] **游戏循环与碰撞失败逻辑**
  - [x] 实现简单的游戏状态管理（如 `Playing` / `GameOver`）。
  - [x] 在飞机与障碍物发生碰撞时，触发失败逻辑：停止控制、播放反馈效果（如粒子、声音）或弹出 UI。
  - [x] 添加重新开始功能（如按键触发或 UI 按钮），重置飞机位置 / 速度并重新生成地图。

- [x] **UI 与反馈**
  - [x] 创建一个简单的 UI 画布，显示当前存活时间 / 距离或得分。
  - [x] 在 Game Over 时显示失败提示与重开选项。
  - [ ] 根据需要添加基础音效（发动机声、碰撞声）和简单的视觉特效。

- [ ] **整理与测试**
  - [ ] 整理脚本和预制体目录结构（如 `Scripts/`, `Prefabs/`, `Scenes/`）。
  - [ ] 在 Unity 编辑器中多次运行游戏，验证：控制手感、随机地图、碰撞检测和重开流程均正常。
  - [ ] 确保控制台无未处理的错误或异常。
