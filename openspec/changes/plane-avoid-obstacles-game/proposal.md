## 标题

开发一个 3D 飞机躲避障碍物小游戏（键盘 ASWD 控制、拟真飞行手感、随机方块地图）

## 背景 / 为什么要做

- 这是一个用于练习 Unity 飞行控制、刚体物理和程序化关卡生成的教学 / Demo 项目。
- 希望玩家通过简单的键盘操作，就能体验到有“重量感”和“魔法力/推力”反馈的飞行操控，而不是简单的平移运动。
- 通过随机生成由立方体（Cube）构成的地图，让每次游戏都有不同的障碍布局，提高可玩性和重玩价值。

## 目标 / 要做什么

- 在当前 Unity 项目中实现一个可玩的飞机躲避障碍物游戏 Loop：
  - 玩家控制一架预制体飞机，在 3D 空间中持续向前飞行。
  - 可以使用键盘 `W/A/S/D` 来控制俯仰 / 横滚 / 偏航 / 推力，使玩家感到飞机具有重量、惯性和推力反馈（“魔法力”）。
  - 使用 `Cube` 搭建一张随机地图：包括随机高度和宽度的障碍走廊 / 墙体 / 柱子等，让玩家在其中穿梭躲避。
  - 检测飞机与障碍物的碰撞，触发失败反馈（如爆炸效果 / 重置关卡 / 显示失败 UI）。
- 提供基础的游戏 UI：
  - 显示当前生存时间 / 距离或得分。
  - 提供重新开始（Restart）入口。
- 所有核心逻辑使用 Unity C# 脚本实现，方便后续扩展。

## 非目标

- 不追求复杂的美术表现（低多边形 / 简单材质即可）。
- 不实现完整的关卡选择、存档系统或复杂的菜单系统。
- 不实现联网、排行榜等在线功能。
- 暂不考虑移动端适配，仅面向 PC 键鼠体验。

## 成功标准

- 玩家能够通过 `W/A/S/D` 控制飞机，明显感受到：
  - 飞机有基本的重力和重量感。
  - 推力变化（“魔法力”）会影响上升 / 下降和速度。
  - 控制时存在一定惯性和回正行为，而不是立即停止。
- 每次开始游戏时，场景中都会生成一组基于 `Cube` 的随机障碍地图，布局存在明显差异。
- 飞机与障碍物发生碰撞时，会触发清晰的失败反馈，并可以重开游戏。
- 在 Unity 编辑器中，游戏可一键 Play 运行，无控制台报错。
