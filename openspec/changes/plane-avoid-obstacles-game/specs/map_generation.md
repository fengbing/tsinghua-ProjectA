## 名称

Cube 随机地图与障碍物生成

## 概要

使用 Unity 的 `Cube` 几何体程序化生成一张具有随机高度和宽度特征的 3D 地图，为飞机提供躲避障碍物的飞行通道。地图在每次开始游戏或重新开始时随机化，保证关卡多样性。

## 详细需求

- 基础地图结构：
  - 地图由若干段（Segment）组成，每段是由多个 `Cube` 构成的障碍区域。
  - 地图沿着飞机的前进方向延伸（例如世界 Z 轴正方向）。
  - 每段地图具有可配置的长度、宽度和高度范围。
- 随机生成规则：
  - 使用伪随机算法（可选地支持传入随机种子）生成障碍布局。
  - 通过 `Cube` 创建墙体、立柱、横梁等障碍物，形成需要玩家精确操控才能通过的通道。
  - 在每个段落中至少保留一条连通的飞行通路，避免完全封死无法通关。
  - 支持在 Inspector 中配置：地图总长度（段数）、单段宽度 / 高度范围、障碍密度等参数。
- 结构与性能：
  - 建议将所有生成的障碍物作为某个空对象（如 `LevelRoot`）的子节点，便于整体管理与清理。
  - 提供重置 / 重新生成接口，用于在玩家重开游戏时清除旧地图并生成新地图。
  - 生成逻辑通过 C# 脚本实现（例如 `CubeMapGenerator`），在场景初始化或游戏开始时调用。

## 验收标准

- 每次进入游戏或重开游戏时，地图中 Cube 障碍物的布局存在明显差异。
- 玩家可以观察到由 Cube 构成的障碍走廊 / 墙体 / 柱子，而不是完全空旷的场景。
- 飞机沿着前进方向飞行时，总能找到一条可行通路，而不是被完全堵死。
- 地图生成和重置过程不会在控制台输出未处理的错误或异常。
