## 名称

飞机控制与物理手感（ASWD + 重量感 + 魔法力）

## 概要

通过键盘 `W/A/S/D` 控制飞机的姿态和推力，基于 Unity `Rigidbody` 物理系统实现具有重量、重力和惯性的飞行手感，同时提供可调参数用于快速迭代体验。

## 详细需求

- 飞机预制体必须包含：
  - `Rigidbody` 组件，启用重力（或通过脚本模拟重力效果），设置合理质量与阻力。
  - 控制脚本（例如 `PlaneController`），仅通过对 `Rigidbody` 施加力 / 力矩或修改速度来驱动，不直接使用 `transform.Translate` 进行位移。
- 键盘控制映射（可在脚本中以变量形式配置，便于后续扩展到 Input System）：
  - `W/S`：控制俯仰角（Pitch），使飞机机头上仰 / 下俯。
  - `A/D`：控制横滚（Roll）或偏航（Yaw），用于左右躲避。
  - 可选：通过某个键或组合（如 `LeftShift` 或持续按 `W`）调节推力强度（“魔法力”），影响当前前进速度或上升能力。
- 物理与手感：
  - 飞机在无输入时，仍然受到重力和空气阻力影响，逐渐下坠或减速。
  - 输入控制时，动作应有一定延迟和过渡，不应瞬间到达目标姿态。
  - 需要可调节的参数（如 `thrustPower`, `rollSpeed`, `pitchSpeed`, `maxSpeed`, `drag`），暴露为 `public` 或 `[SerializeField]` 字段，方便在 Inspector 中调节。
  - 允许设计“魔法力”概念，例如在一定时间内提供额外升力或推力，并在 UI 或数值上有所体现（如能量条）。

## 验收标准

- 在场景中放置飞机预制体并运行游戏时：
  - 不按任何键时，飞机会在重力影响下逐渐发生姿态 / 高度变化，而不是完全静止在空中。
  - 按下 `W/S/A/D` 时，飞机以平滑方式改变姿态和方向，具有惯性，不会瞬间停下。
- 调整控制脚本中的参数可以明显改变飞机控制的敏锐度和重量感。
- 所有控制与物理计算通过 `Rigidbody`（或等效物理方式）完成，而非直接修改 Transform 位置。
