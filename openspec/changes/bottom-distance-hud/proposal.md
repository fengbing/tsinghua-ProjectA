## Why

玩家在飞行时需要快速感知与前方障碍或目标的剩余距离；纯数字 HUD 不够醒目。在屏幕底部增加一条「高级」距离带，用颜色、渐变模糊与呼吸式动效在视野边缘持续提示危险程度，能减轻认知负担并增强紧张感。

## What Changes

- 在屏幕底部区域新增一条**距离提示带**（不占满全屏高度，以底部条带形式呈现）。
- 视觉：**半透明**、**渐变**、带**模糊**（毛玻璃感），光感从**屏幕水平中心向左右两侧衰减发散**（非均匀整条同色）。
- **颜色与距离阈值**（可配置默认值）：
  - 距离 **> 300 m**：以**白色**系为主（安全区）。
  - 距离 **≤ 150 m**：过渡为**黄色**系（警戒）。
  - 距离 **≤ 50 m**：**红色**系（高危）。
  - 在相邻阈值之间使用平滑插值，避免硬切。
- **呼吸灯**：整体亮度或透明度随时间周期性脉动（节奏舒缓、可配置周期），强化「活着的」提示感。
- **数据来源**：由现有或新增的游戏逻辑提供「当前显示距离」（米）；若项目尚无统一接口，则定义一个轻量 API（如 Scriptable 事件或单例服务）供 UI 订阅。

## Capabilities

### New Capabilities

- `distance-hud`：底部距离提示带的呈现、阈值配色、呼吸动画、渐变模糊视觉，以及与游戏距离数据的绑定约定。

### Modified Capabilities

- （无）`openspec/specs/` 下暂无已归档能力；本变更为独立新能力。

## Impact

- **Unity**：Canvas / UI Toolkit 或 Image + Material；若使用 URP/HDRP 可能需要 **UI Blur**（如 GrabPass 替代方案、自定义 Renderer Feature 或第三方方案），需在目标渲染管线下验证性能与兼容性。
- **脚本**：新增 UI 控制器（如 `DistanceHudView`）与可选的 `IDistanceProvider` 或现有飞行/游戏管理器上的距离输出。
- **场景 / 预制体**：主游戏 Canvas 或 HUD 预制体增加该组件与引用。
