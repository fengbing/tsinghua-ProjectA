## Context

项目为 Unity 3D 飞机躲避障碍玩法，已有相机跟随与飞机控制等脚本。玩家需要**不盯数字**也能感知「离危险有多近」。本设计在**屏幕底部**增加一条装饰性、信息性兼顾的距离带，与 `distance-hud` 能力对应。

约束：需在当前渲染管线（以 URP 为常见默认）下可落地；移动端非首要目标时可接受 PC 优先。

## Goals / Non-Goals

**Goals:**

- 用**颜色 + 动效**表达与障碍的剩余距离（米），阈值默认 **300 / 150 / 50**。
- 视觉：**半透明**、**水平中心亮、向两侧衰减**的渐变；整体带**呼吸式**周期脉动。
- 实现上优先**可维护**：距离来源与 UI 解耦（接口或事件），便于接真实测距逻辑或占位数据。

**Non-Goals:**

- 不规定具体「测距几何」（例如是到最近障碍、到航线终点还是到下一门洞），仅规定 UI 接收的标量「显示距离」。
- 不要求与某第三方后处理强绑定；模糊若管线成本过高，允许降级为「高斯模糊感」的**多层渐变 + 柔边**而不阻塞交付。

## Decisions

1. **UI 技术栈：Unity UI (uGUI)**  
   - **理由**：与现有 Canvas/HUD 扩展一致，美术与布局迭代快。  
   - **备选**：UI Toolkit — 若项目全面迁移可再迁，本阶段不引入双栈。

2. **「从中间向两边发散」**  
   - 使用 **水平方向** alpha / 颜色权重：中心（屏幕 x=0.5）权重最高，向左右衰减至接近透明。可用 **Image + 横向渐变纹理**，或 **Shader** 中 `abs(uv.x - 0.5)` 驱动。  
   - **备选**：径向渐变以中心为圆心 — 与用户描述的「左右发散」不符，故不采用。

3. **颜色与阈值**  
   - **> 300 m**：白色系（可略带冷灰以免刺眼）。  
   - **150 m < d ≤ 300 m**：白 → 黄插值。  
   - **50 m < d ≤ 150 m**：黄 → 红插值。  
   - **≤ 50 m**：红色系为主。  
   - 插值在**连续距离**上平滑，避免阈值处闪烁；若单帧距离跳变过大，可对**用于染色的标量**做短时平滑（可选，见 Open Questions）。

4. **呼吸灯**  
   - 对**整体不透明度**或**叠加亮度**施加 `sin(time * ω)`，周期默认约 **2–4 s**（Inspector 可配），幅度适中以免眩晕。

5. **渐变模糊**  
   - **首选**：若目标 Unity/URP 版本具备可接受的 **UI 模糊**（自定义 Blit、Renderer Feature、或官方/社区方案），在底部条带区域应用**局部**模糊，与渐变遮罩相乘。  
   - **降级**：无模糊时，用**多层半透明 Image** + **柔边渐变**模拟毛玻璃层次，仍满足「半透明渐变」产品描述。

6. **距离数据**  
   - 定义 `IDistanceHudSource`（或等价）：`float GetDisplayDistanceMeters()` 或事件 `OnDistanceChanged(float meters)`。  
   - 飞机或关卡脚本实现该接口或在管理器上转发，HUD 仅负责显示。

## Risks / Trade-offs

- **[风险] 真 UI 模糊在 URP 上实现成本与版本差异大** → **缓解**：设计允许降级为无 GrabPass 的纯渐变方案；先验收颜色与呼吸，再迭代模糊。  
- **[风险] 高帧率下 sin 脉动与距离跳变叠加造成刺眼** → **缓解**：限制呼吸幅度上限；距离变色用 Lerp 跟随。  
- **[权衡] 底部条带可能遮挡少量画面** → **缓解**：条带高度可配置，默认较扁（如屏高 8%–12%）。

## Migration Plan

- 新增 HUD 预制体或 Canvas 子节点，**默认关闭**或 **Editor 内默认开启**以便联调；接入真实距离后于主场景启用。  
- 回滚：移除 Canvas 子物体或禁用组件即可，不破坏飞行与碰撞逻辑。

## Open Questions

- 「显示距离」的精确定义（最近障碍、前方射线、剩余航线长度）由哪一模块提供 — 需在实现阶段与玩法脚本对齐。  
- 是否需要 **HDR/泛光** 强化红色警戒 — 若性能允许可作为视觉增强项，非本设计必选。
