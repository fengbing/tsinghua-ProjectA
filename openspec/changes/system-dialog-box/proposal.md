## Why

当前项目缺少可控的系统对话框能力，无法在关键时刻稳定地向玩家传达剧情、提示或任务信息。需要一个统一的底部对话框组件，支持按需显示、逐字展示文本与句级语音播放，以提升信息传达清晰度和沉浸感。

## What Changes

- 新增一个可复用的系统对话框 UI，固定在画面底部，宽度覆盖整个游戏画面，背景为黑色矩形。
- 支持按需显示/隐藏对话框，不需要时不占用可见 UI。
- 支持文本在对话框中居中显示，文字为白色。
- 支持逐字打字机效果（字符一个一个出现），可配置速度并可在需要时跳过到整句完成。
- 支持每句话绑定并播放对应音频，句子切换时自动处理当前音频与下一句音频。
- 提供可被游戏流程调用的接口（例如播放一组对话句子、监听句子完成/整段完成）。

## Capabilities

### New Capabilities
- `system-dialog-ui`: 底部全宽黑色系统对话框的显示、隐藏、样式与文本呈现能力。
- `dialog-typewriter-and-voice`: 对话文本逐字显示与每句话音频播放的同步控制能力。

### Modified Capabilities
- (none)

## Impact

- 受影响代码：`Assets/Scripts` 下新增对话框控制脚本，必要时对现有教程/剧情触发脚本增加调用点。
- 受影响资源：新增或复用 UI 预制体、音频资源引用配置（`AudioClip`）。
- 运行时影响：新增 UI 更新与协程/计时逻辑，需保证在暂停、场景切换和重复触发场景下行为稳定。
